# AstrBot 插件文档：mepin

## 插件简介

`mepin` 是一个轻量级娱乐型插件，用于在群聊中为指定用户的消息增添趣味性。当被监控的用户发送消息时，插件会以设定的概率自动回复“没品”，营造轻松幽默的群聊氛围。该功能默认关闭，仅机器人管理员可启用或配置。
本插件完全本地运行，不进行任何网络请求，不收集用户隐私数据，不依赖外部服务。

   插件加载时，若无配置则使用默认值：
   - 功能开关：`false`
   - 触发概率：`10`（即 10%）
   - 监控用户列表：`[]`

## 使用方法

#### 1. 开启/关闭“没品”功能
```text
/mepin toggle
```
- **作用**：切换功能开关状态。
- **回复示例**：
  - `✅ “没品”功能已开启。`
  - `❌ “没品”功能已关闭。`

#### 2. 设置触发概率
```text
/mepin setprob n
```
- **参数**：`n` 为 0–100 的整数，表示触发概率百分比。
- **示例**：
  - `/mepin setprob 5` → 每 20 条消息约触发 1 次。
  - `/mepin setprob 100` → 每条消息都回复“没品”。
  - `/mepin setprob 150` → **错误**：概率超出范围，系统拒绝并提示。
- **回复示例**：`🎯 “没品”触发概率已设置为 25%。`

#### 3. 查看监控用户列表
```text
/mepin list
```
- **作用**：列出当前所有被监控的用户 ID。
- **回复示例**：
  ```
  👥 当前监控用户列表（共 2 人）：
  - 123456789
  - 987654321
  ```

#### 4. 添加监控用户
```text
/mepin add 123456789
```
- **参数**：`<user_id>` 为平台唯一用户标识（如 QQ 号、Telegram 用户 ID）。
- **注意**：重复添加无效果，系统自动去重。
- **回复示例**：`➕ 用户 123456789 已加入监控列表。`

#### 5. 移除监控用户
```text
/mepin remove 123456789
```
- **参数**：`<user_id>` 为要移除的用户标识。
- **注意**：若用户不在列表中，系统提示“未找到”。
- **回复示例**：
  - `➖ 用户 123456789 已从监控列表中移除。`
  - `❓ 用户 123456789 不在监控列表中。`

#### 6. 手动测试
```text
/mepin_test
```
- **注意**：无参数，用于测试能否正常发送“没品”。

### 使用流程示例

1. 管理员执行 `/mepin toggle` → 开启功能。
2. 执行 `/mepin setprob 15` → 设置 15% 触发率。
3. 执行 `/mepin add 123456789` → 监控某用户。
4. 该用户在群聊中发送消息：“今天吃饭了吗？” → 插件有 15% 概率回复：“没品”。
5. 后续可随时执行 `/mepin list` 查看状态，或 `/mepin remove 123456789` 停止监控。


## 注意事项

- ✅ **仅管理员可操作**：所有配置命令均需管理员权限，防止滥用。
- ✅ **无网络请求**：插件完全离线运行，不访问任何外部 API、网站或服务。
- ✅ **无隐私收集**：仅存储用户 ID（用于识别），不记录消息内容、时间、聊天记录。
- ✅ **兼容性**：支持所有 AstrBot 支持的平台（QQ、Telegram、Lark 等），通过平台适配器自动识别用户 ID。
- ⚠️ **概率非精确**：触发为随机事件，非计数器，长期观察可能偏离设定概率。
- ❌ **禁止滥用**：请勿用于恶意刷屏、骚扰或针对特定用户进行网络暴力，违反此原则将被社区抵制。

> 本插件仅为趣味互动工具，建议在轻松友好的群环境中使用，尊重每位成员的体验。
