# AstrBot 插件文档：mepin

## 插件简介

`mepin` 是一个轻量级娱乐型插件，用于在群聊中为指定用户的消息增添趣味性。当被监控的用户发送消息时，插件会以设定的概率自动回复“没品”，营造轻松幽默的群聊氛围。该功能默认关闭，仅机器人管理员可启用或配置，确保使用可控、无干扰。

本插件完全本地运行，不进行任何网络请求，不收集用户隐私数据，不依赖外部服务，符合 AstrBot 安全与隐私规范。

## 功能说明

- **概率触发回复**：当被监控用户发言时，插件以设定概率（默认 10%）自动回复“没品”。
- **管理员控制**：仅机器人管理员可开启/关闭功能、修改概率、管理监控列表。
- **灵活监控列表**：支持添加或移除任意用户（通过 QQ 号、Telegram ID 等平台唯一标识）。
- **状态可视化**：提供命令查看当前监控用户列表与功能开关状态。
- **无干扰设计**：不响应非监控用户消息，不影响正常聊天流程。

## 插件流程

`mepin` 插件的工作流程遵循 AstrBot 的事件驱动架构，具体逻辑如下：

1. **初始化阶段**  
   插件加载时，从本地存储读取配置（功能开关状态、触发概率、监控用户列表），若无配置则使用默认值：
   - 功能开关：`false`
   - 触发概率：`10`（即 10%）
   - 监控用户列表：`[]`

2. **事件监听阶段**  
   插件注册一个全局消息监听器（非命令），监听所有群聊消息（`GROUP_MESSAGE`）。该监听器优先级为 `0`，不影响其他插件执行。

3. **消息处理阶段**  
   当收到一条群消息时，执行以下判断：
   - 若功能开关为 `false` → 直接忽略，不执行后续逻辑。
   - 若消息发送者不在监控列表中 → 忽略。
   - 若消息发送者在监控列表中 → 生成一个 0–100 的随机数。
     - 若随机数 ≤ 设置的概率值（如 10）→ 生成回复消息 `"没品"` 并通过 `event.send()` 主动发送。
     - 否则 → 忽略，不作任何响应。

4. **命令处理阶段**  
   插件注册五个命令，仅允许管理员使用（通过 `@filter.permission_type(PermissionType.ADMIN)` 限制）：
   - `mepin toggle`：切换功能开关状态。
   - `mepin setprob <n>`：设置概率值（0–100），超出范围则拒绝。
   - `mepin list`：返回当前监控用户列表。
   - `mepin add <user_id>`：将指定用户 ID 加入监控列表（去重）。
   - `mepin remove <user_id>`：从监控列表中移除指定用户。

5. **持久化存储**  
   所有配置（开关、概率、用户列表）通过 AstrBot 的 KV 存储机制（`self.put_kv_data()`）保存在 `data/plugin_data/astrbot_plugin_mepin/` 目录下，确保重启后配置不丢失。

> ⚠️ 注意：插件不记录任何用户聊天内容，不分析消息语义，仅基于用户 ID 和概率做简单触发，无任何 LLM 调用或外部依赖。

## 使用方法

### 前提条件
- 插件已安装并启用。
- 使用者为机器人管理员（已在 `cmd_config.json` 中配置为 `admins_id`）。

### 命令详解

#### 1. 开启/关闭“没品”功能
```text
/mepin toggle
```
- **作用**：切换功能开关状态。
- **回复示例**：
  - `✅ “没品”功能已开启。`
  - `❌ “没品”功能已关闭。`

#### 2. 设置触发概率
```text
/mepin setprob 25
```
- **参数**：`<n>` 为 0–100 的整数，表示触发概率百分比。
- **示例**：
  - `/mepin setprob 5` → 每 20 条消息约触发 1 次。
  - `/mepin setprob 100` → 每条消息都回复“没品”。
  - `/mepin setprob 150` → **错误**：概率超出范围，系统拒绝并提示。
- **回复示例**：`🎯 “没品”触发概率已设置为 25%。`

#### 3. 查看监控用户列表
```text
/mepin list
```
- **作用**：列出当前所有被监控的用户 ID。
- **回复示例**：
  ```
  👥 当前监控用户列表（共 2 人）：
  - 123456789
  - 987654321
  ```

#### 4. 添加监控用户
```text
/mepin add 123456789
```
- **参数**：`<user_id>` 为平台唯一用户标识（如 QQ 号、Telegram 用户 ID）。
- **注意**：重复添加无效果，系统自动去重。
- **回复示例**：`➕ 用户 123456789 已加入监控列表。`

#### 5. 移除监控用户
```text
/mepin remove 123456789
```
- **参数**：`<user_id>` 为要移除的用户标识。
- **注意**：若用户不在列表中，系统提示“未找到”。
- **回复示例**：
  - `➖ 用户 123456789 已从监控列表中移除。`
  - `❓ 用户 123456789 不在监控列表中。`

#### 6. 手动测试
```text
/mepin_test
```
- **注意**：无参数，用于测试能否正常发送“没品”。

### 使用流程示例

1. 管理员执行 `/mepin toggle` → 开启功能。
2. 执行 `/mepin setprob 15` → 设置 15% 触发率。
3. 执行 `/mepin add 123456789` → 监控某用户。
4. 该用户在群聊中发送消息：“今天吃饭了吗？” → 插件有 15% 概率回复：“没品”。
5. 后续可随时执行 `/mepin list` 查看状态，或 `/mepin remove 123456789` 停止监控。

## 配置说明

本插件无外部配置文件（如 `_conf_schema.json`），所有配置均通过命令动态管理，并持久化至插件私有存储中。配置项包括：

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `enabled` | `bool` | `false` | 是否启用“没品”回复功能 |
| `probability` | `int` | `10` | 触发概率（0–100） |
| `monitored_users` | `list[str]` | `[]` | 被监控用户的平台唯一 ID 列表 |

> 所有配置由插件内部自动管理，用户无需手动编辑文件。

## 注意事项

- ✅ **仅管理员可操作**：所有配置命令均需管理员权限，防止滥用。
- ✅ **无网络请求**：插件完全离线运行，不访问任何外部 API、网站或服务。
- ✅ **无隐私收集**：仅存储用户 ID（用于识别），不记录消息内容、时间、聊天记录。
- ✅ **无副作用**：不修改、删除、读取任何系统文件；不调用 `eval`、`exec`、`os.system` 等危险函数。
- ✅ **无强制互动**：用户可随时移除监控，功能可随时关闭，不影响群聊正常交流。
- ✅ **兼容性**：支持所有 AstrBot 支持的平台（QQ、Telegram、Lark 等），通过平台适配器自动识别用户 ID。
- ⚠️ **概率非精确**：触发为随机事件，非计数器，长期观察可能偏离设定概率。
- ❌ **禁止滥用**：请勿用于恶意刷屏、骚扰或针对特定用户进行网络暴力，违反此原则将被社区抵制。

> 本插件仅为趣味互动工具，建议在轻松友好的群环境中使用，尊重每位成员的体验。
